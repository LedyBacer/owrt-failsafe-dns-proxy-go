# Обзор

[English](../en/overview.md) | **Русский**

## Проблема

На OpenWrt dnsmasq — DNS-сервер для LAN: отвечает клиентам, кэширует ответы и
разрешает локальные имена. Для внешнего DNS он обычно ходит на один адрес —
например локальный `https-dns-proxy`, `stubby` или публичный резолвер.

Когда этот upstream перестаёт отвечать, DNS пропадает на всех устройствах, пока
вы не заметите проблему и не смените настройки. dnsmasq не умеет:

- пробовать upstream-ы в фиксированном порядке приоритета;
- пропускать upstream, который уже признан недоступным;
- возвращаться на предпочитаемый резолвер после восстановления.

Несколько строк `server=` в dnsmasq — не то же самое: это несколько резолверов,
а не строгая схема «основной + резерв» с проверкой состояния и автоматическим
failback.

## Решение

Failsafe DNS Proxy — небольшой локальный daemon, который dnsmasq использует как
единственный upstream. Daemon пересылает запросы на ваши реальные резолверы:

1. **Основной** — предпочитаемый (часто локальный шифрованный DNS).
2. **Резервный** — простой DNS, который должен работать, когда основной путь
   недоступен.

Daemon выбирает работающий upstream с наивысшим приоритетом, переключается в
пределах заданных таймаутов, запоминает недавние отказы и возвращается на
основной только после подтверждения восстановления фоновыми probes.

## Чем проект не является

| Этот проект | Не это |
| --- | --- |
| Слой failover между dnsmasq и upstream-ами | Замена dnsmasq |
| Plain UDP/TCP forwarding на настроенные резолверы | DoH/DoT/DoQ-клиент |
| Выбор upstream с учётом состояния | DNS cache, блокировка рекламы, фильтрация |
| Явная обратимая интеграция с dnsmasq | Тихая перезапись DNS-настроек роутера |

Если вам нужен один стабильный публичный резолвер и вы не используете локальный
DNS proxy, daemon, скорее всего, не нужен — достаточно направить dnsmasq
напрямую на этот резолвер.

## Типичная схема

```text
клиенты LAN
    |
    v
dnsmasq          <- клиенты, кэш, локальные имена
    |
    v
failsafe-dns-proxy 127.0.0.1:5359
    |-- priority 10 -> https-dns-proxy 127.0.0.1:5054
    `-- priority 20 -> резерв 77.88.8.8:53
```
